

📁 用户相关
User（用户）
存储所有注册玩家的账号信息：

用户名、密码（加密）、昵称、头像
等级、经验值
注册时间、最后登录时间
📁 房间相关
房间 (Room)
每创建一个游戏房间就有一条记录：

房间名称（如"测试房间"）
谁创建的、谁是主持人
最大人数、当前人数
状态：waiting(等待中) / playing(游戏中) / closed(已关闭)
房间密码（如果设了的话）
房间玩家 (RoomPlayer)
记录"谁加入了哪个房间"：

哪个用户、哪个房间
角色：host(主持人) / human_player(玩家)
玩家序号（0, 1, 2...）
加入时间、离开时间
HostConfig（主持人配置）
主持人在开始游戏前的所有设置：

AI配置：用哪个AI（DeepSeek/OpenAI）、API密钥、端点地址
游戏规则：主持人写的规则文本
人数配置：总主体数、人类玩家数、AI玩家数
时间配置：每回合决策时限（分钟）、超时策略
是否完成初始化
📁 游戏进行中
游戏会话 (GameSession)
一局游戏的核心状态：

属于哪个房间
当前第几回合
回合阶段：decision(决策中) / inference(AI推演中) / result(看结果)
决策截止时间
gameState：完整的游戏状态JSON（所有主体的资金、属性、卦象等）
玩家行动 (PlayerAction)
玩家每回合提交的决策：

哪个玩家、第几回合
决策文本（玩家输入的内容，如"投资研发"）
选择的选项ID
主持人是否修改过
提交时间
AI的响应结果
TemporaryEvent（临时事件）
游戏中的多回合持续事件：

事件类型（正面/负面/中性）
事件内容描述
持续几个回合
当前进度
Trade（交易）
玩家之间的交易请求：

谁发起、发给谁
交易什么资源
状态：pending(待处理) / accepted(已接受) / rejected(已拒绝)
过期时间
📁 存档与任务
游戏存档 (GameSave)
手动或自动保存的游戏进度：

存档名称、描述
是否自动存档
saveData：完整的游戏快照JSON（可以恢复到这个状态）
任务 (Task)
游戏内的任务系统：

任务标题、描述
类型、难度
完成要求、奖励
状态：进行中/已完成
TaskProgress（任务进度）
每个玩家的任务完成情况

📁 AI与策略
AIConversation历史
与AI的对话记录（用于跨回合记忆）：

哪局游戏、第几回合
角色：user(发给AI的) / assistant(AI回复的)
对话内容
token数量
战略 (Strategy)
玩家保存的个人策略：

策略名称、描述
策略数据
胜率、平均分、使用次数
简单总结
表	一句话说明
User	谁注册了
房间	有哪些房间
房间玩家	谁在哪个房间
HostConfig	主持人怎么配置的
游戏会话	游戏进行到哪了
玩家行动	玩家做了什么决策
TemporaryEvent	正在发生什么事件
Trade	谁和谁在交易
游戏存档	保存的进度
任务	有什么任务
AI历史	AI记住了什么
战略	玩家的策略库
